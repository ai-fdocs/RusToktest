# leptos-graphql

## Назначение
`leptos-graphql` — клиент GraphQL для Leptos-приложений с поддержкой запросов, мутаций и подписок.

## Что делает
- Выполняет GraphQL-запросы к API сервера.
- Предоставляет реактивные хуки для работы с GraphQL в Leptos.
- Поддерживает кэширование результатов запросов.
- Обрабатывает ошибки и состояния загрузки.
- Поддерживает аутентификацию через заголовки.

## Как работает (простыми словами)
1. Клиент инициализируется с базовым URL API и опциональным токеном.
2. Компоненты используют хуки (`use_query`, `use_mutation`) для выполнения запросов.
3. Результаты запросов являются реактивными сигналами Leptos.
4. При изменении переменных запрос автоматически перезапускается.
5. Ошибки и состояние загрузки доступны через отдельные сигналы.

## Ключевые компоненты
- `client/` — HTTP-клиент для GraphQL-запросов.
- `hooks/` — хуки `use_query`, `use_mutation`, `use_subscription`.
- `cache/` — кэширование результатов запросов.
- `error/` — обработка ошибок GraphQL и сети.

## Кому нужен
Для всех фронтенд-приложений на Leptos, взаимодействующих с GraphQL API:
- `apps/admin` — вся работа с данными через GraphQL.
- `apps/storefront` — получение каталога, оформление заказов.
